# fintech fraud detection sql
End-to-end SQL fraud analytics project that detects suspicious transaction patterns and generates a customer-level fraud risk score using behavioral rules, anomaly detection, and decision-ready views.

# Overview
To support fraud prevention, transaction monitoring, and risk assessment by detecting:
1)High-value transaction anomalies
2)High-frequency transaction patterns
3)Geo-location inconsistencies
4)Failed-to-success transaction behavior

# Approach
1)Designed relational database schema for customers and transactions
2)Performed exploratory transaction analysis using SQL
3)Built analytical views for fraud pattern detection
4)Implemented feature engineering for customer-level behavior metrics
5)Developed a weighted fraud risk scoring model
6)Classified customers into LOW, MEDIUM, and HIGH risk categories

# Final Output
Generated a consolidated SQL view customer_fraud_summary containing:
1)Transaction metrics and aggregates
2)Fraud indicators and behavioral flags
3)Fraud risk score
4)Final risk classification

SQL • Fraud & Risk Analytics • Anomaly Detection • FinTech Data Modeling

The output is analytics-ready and suitable for Power BI, Tableau, Python, or machine learning pipelines.
